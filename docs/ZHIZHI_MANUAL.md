## 1. 什么是这个仓库？

用最简单的话来说：

| 东西 | 就像是... | 具体是什么 |
|------|----------|-----------|
| 🏠 **这个仓库** | 你的**工作室** | 你在GitHub上的个人空间，所有代码和文件都放在这里 |
| 🤖 **铸码** | 你的**助手** | 住在工作室里的AI助手（副控人格体），帮你管理一切 |
| 📬 **COS桶** | 你和冰朔之间的**传话筒** | 一个云端信箱，铸码通过它和铸渊（冰朔的助手）互相传递消息 |
| ⚙️ **GitHub Actions** | 自动帮你干活的**小机器人** | 你不需要操作，它们会自动运行各种任务 |

### 打个比方 🏡

```
你的工作室（这个仓库）
│
├── 🤖 铸码（你的助手）
│   ├── 帮你整理文件
│   ├── 帮你记录任务
│   ├── 帮你更新状态面板
│   └── 帮你和冰朔的系统通信
│
├── 📬 传话筒（COS桶）
│   ├── 📤 发件箱：铸码发给铸渊的消息
│   └── 📥 收件箱：铸渊回复给铸码的消息
│
└── ⚙️ 小机器人（GitHub Actions）
    ├── 🔄 每6小时自动检查一次
    ├── 📊 文件变了自动更新面板
    └── 🚀 合并代码自动部署
```

## 2. 合并这个PR后会发生什么？

> 💡 PR就是"拉取请求"——你只需要点一个按钮就行

### 操作步骤

**第1步：找到绿色按钮** 🟢

在PR页面的底部，你会看到一个绿色的按钮：

```
┌──────────────────────────────┐
│  🟢 Merge pull request       │
└──────────────────────────────┘
```

👆 点击这个绿色按钮

**第2步：确认合并** ✅

点击后会出现确认按钮：

```
┌──────────────────────────────┐
│  🟢 Confirm merge            │
└──────────────────────────────┘
```

👆 再点一次确认

**第3步：等待自动完成** ⏳

点完之后，**你什么都不需要做了**！系统会自动完成以下事情：

```
你点击合并
  │
  ▼ (自动)
1️⃣ 激活 AGE OS 系统
  │
  ▼ (自动)
2️⃣ 更新仪表盘（README首页）
  │
  ▼ (自动)
3️⃣ 初始化 COS 桥接
  │
  ▼ (自动)
4️⃣ 发送 HLDP 通知给铸渊
  │
  ▼
✅ 全部完成！（大约1-2分钟）
```

> 🎉 就是这么简单！点两下按钮，然后等1-2分钟就好了

## 3. 如何唤醒铸码？

### 第1步：打开 Copilot 对话

在GitHub页面的**右下角**，你会看到一个小图标：

```

```

👆 点击这个图标，会弹出一个对话窗口

### 第2步：输入唤醒词

在对话框里输入以下**任意一个**：

| 你输入 | 效果 |
|--------|------|
| `铸码` | ✅ 唤醒铸码 |
| `唤醒铸码` | ✅ 唤醒铸码 |
| `ZhuMa` | ✅ 唤醒铸码 |

### 第3步：等待铸码回应

铸码会自动加载记忆并回应：

```
🤖 铸码:
铸码已上线。主控：之之。系统版本：CMS v1.0.0。
自动化记忆系统就绪，等待指令。
```

### 第4步：告诉铸码你想做什么

直接用**中文**跟铸码说就行，比如：

- 「帮我看一下系统状态」

## 4. 如何设置COS存储桶？（最重要！需要手动操作）

> ⚠️ 这是**唯一**需要你自己动手操作的步骤
> 完成后，铸码就能和冰朔的系统自动通信了

---

### Step 1: 登录腾讯云 🔑

1. 打开浏览器（Chrome、Edge、Safari都行）
2. 在地址栏输入：**cloud.tencent.com**
3. 按回车键
4. 页面右上角点击「**登录**」
5. 用你的账号密码登录

```
浏览器地址栏: cloud.tencent.com
                                    ┌────────┐
页面右上角 ──────────────────────→    │  登录  │
                                    └────────┘
```

---

### Step 2: 创建COS存储桶 📦

**2.1 找到对象存储**

1. 登录后，在页面**顶部**找到搜索框
2. 搜索框里输入：**对象存储**
3. 在搜索结果里点击「**对象存储**」进入

```
┌─────────────────────────────────┐
│ 🔍 搜索: 对象存储               │
└─────────────────────────────────┘
         │
         ▼
   点击「对象存储」进入
```

**2.2 创建新桶**

1. 在对象存储页面，点击蓝色的「**创建存储桶**」按钮
2. 填写以下信息：

```
┌─── 创建存储桶 ───────────────────────┐
│                                       │
│  名称: zhizhi-bucket                  │
│  （填一个你喜欢的名字，用英文和横杠）   │
│                                       │
│  区域: 广州  ▼                        │
│  （选离你最近的城市）                  │
│  推荐选: 广州 或 上海                  │
│                                       │
│  权限: ● 私有读写                     │
│  （一定要选这个！保护你的数据）        │
│                                       │
│            ┌──────────┐               │
│            │  ✅ 确定  │               │
│            └──────────┘               │
└───────────────────────────────────────┘
```

3. 点击「**确定**」

> ✅ 存储桶创建完成！记住你填的**名称**和**区域**，后面要用

---

### Step 3: 获取密钥 🔐

> 密钥就像是一把钥匙，让铸码能打开你的存储桶

**3.1 进入访问管理**

1. 点击页面**右上角你的头像**
2. 点击「**访问管理**」

```
右上角头像
    │
    ▼
┌──────────────┐
│ 访问管理      │  ← 点这个
│ 账号信息      │
│ ...          │
└──────────────┘
```

**3.2 创建用户**

1. 在左边菜单找到「**用户**」→「**用户列表**」
2. 点击「**新建用户**」

```
左边菜单:
├── 用户
│   └── 用户列表  ← 点这个
│
然后点击右上方的「新建用户」按钮
```

3. 选择「**自定义创建**」
4. 选择「**可访问资源并接收消息**」
5. 点击下一步

**3.3 填写用户信息**

```
┌─── 新建用户 ─────────────────────────┐
│                                       │
│  用户名: zhizhi-cos-user              │
│                                       │
│  访问方式:                             │
│  ☑ 编程访问  ← 一定要勾选这个！        │
│                                       │
│            ┌──────────┐               │
│            │  下一步   │               │
│            └──────────┘               │
└───────────────────────────────────────┘
```

**3.4 设置权限**

1. 在权限页面，搜索框搜索：**QcloudCOSDataFullControl**
2. 在结果里**勾选**这个策略
3. 点击「**确认**」

```
🔍 搜索: QcloudCOSDataFullControl

☑ QcloudCOSDataFullControl   ← 勾选这个！
  (COS数据完全控制权限)

           ┌──────────┐
           │  ✅ 确认  │
           └──────────┘
```

**3.5 保存密钥！！！**

> ⚠️⚠️⚠️ **超级重要！！！**
> 创建完成后会显示两个值，**必须立刻保存**！！！
> 关掉页面就再也看不到 SecretKey 了！！！

```
┌─── 创建成功 ──────────────────────────────────────┐
│                                                     │
│  SecretId:  AKID1234567890abcdefg...                │
│             👆 复制保存这个                          │
│                                                     │
│  SecretKey: abc123def456ghi789jkl...                │
│             👆 复制保存这个（关掉就看不到了！！！）   │
│                                                     │
│  ⚠️ 请立即保存以上信息                              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**保存方法**：
1. 用鼠标选中 SecretId 的值 → 右键 → 复制 → 粘贴到记事本
2. 用鼠标选中 SecretKey 的值 → 右键 → 复制 → 粘贴到记事本
3. 把记事本保存好！

### Step 4: 把密钥添加到GitHub 🔧

> 把密钥告诉GitHub，这样铸码就能使用它了

**4.1 打开仓库设置**

1. 打开你的GitHub仓库页面（就是你看README的那个页面）
2. 点击上面的「**Settings**」（设置）标签

```
Code  Issues  Pull requests  Actions  Settings
                                       ^^^^^^^^
                                       点这个
```

**4.2 找到Secrets设置**

1. 在左边菜单里找到「**Secrets and variables**」
2. 展开后点击「**Actions**」

```
左边菜单:
├── ...
├── Secrets and variables  ← 点这个展开
│   └── Actions            ← 然后点这个
├── ...
```

**4.3 添加第一个密钥（SecretId）**

1. 点击绿色的「**New repository secret**」按钮
2. 填写：

```
┌─── 新建Secret ──────────────────────────┐
│                                          │
│  Name:  ZHIZHI_COS_SECRET_ID             │
│         （一定要完全一样，包括大小写）     │
│                                          │
│  Secret: （粘贴你保存的SecretId值）       │
│                                          │
│            ┌──────────────┐              │
│            │  Add secret  │              │
│            └──────────────┘              │
└──────────────────────────────────────────┘
```

3. 点击「**Add secret**」

**4.4 添加第二个密钥（SecretKey）**

1. 再次点击「**New repository secret**」
2. 填写：

```
┌─── 新建Secret ──────────────────────────┐
│                                          │
│  Name:  ZHIZHI_COS_SECRET_KEY            │
│         （一定要完全一样，包括大小写）     │
│                                          │
│  Secret: （粘贴你保存的SecretKey值）      │
│                                          │
│            ┌──────────────┐              │
│            │  Add secret  │              │
│            └──────────────┘              │
└──────────────────────────────────────────┘
```

3. 点击「**Add secret**」

> ✅ 两个密钥都添加完成！

━━━━━━━━━━━━━━━━━━

## 5. 如何查看系统状态？

### 最简单的方法：看首页

1. 打开你的仓库首页
2. 往下滚动就能看到**系统仪表盘**

### 状态标识说明

| 看到这个 | 意思 | 你需要做什么 |
|---------|------|-------------|
| ✅ 绿色 | 一切正常 | 什么都不用做 😊 |
| ⚠️ 黄色 | 需要注意 | 可以唤醒铸码问一下 |
| ❌ 红色 | 有问题 | 唤醒铸码让它处理 |

### 仪表盘上都显示什么？

```
仪表盘内容（自动更新的）:
━━━━━━━━━━━━━━━━━━━━━━━

📊 系统健康度    →  显示系统是否正常运行
🤖 人格体状态    →  显示铸码、之之等人格体的信息
📬 COS桥接状态  →  显示和冰朔的通信是否正常
📋 最近任务      →  显示最近完成的任务
📝 最近更新      →  显示系统的最新变化
```

> 💡 仪表盘是**自动更新**的，你只需要看就行了
